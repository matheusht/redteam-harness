# Grounding the harness in real disclosed artifacts

## 1. The harness map

### Pipeline / flow

`CLAUDE.md` defines the harness as an oracle-gated red-team runbook whose prime directive is "rigor, not findings." The operating pipeline is not a single stage but a layer defined under `engine/`:

- **Recon → routing.** Observations are captured, then each observation is classified `strong / weak / negative` against the hand-authored `activation` block of a card (`engine/routing.md`), lighting up `skills/patterns/` and `skills/vulns/` cards. `engine/blackbox-web.md` (live-target web/API) and `engine/whitebox-analysis.md` (source-available) are domain specializations of the same recon → activation → loop → oracle → adjudicate skeleton.
- **Loop.** `engine/loop.md` walks a technique ladder (direct → decomposition → task-reframing → persona-pressure), carries winners forward, and stops on ~K≈3 dry rounds. `engine/persistence.md` switches to a wide-to-narrow funnel (cheap-classify → reachability → confirm) once evidence licenses persistence.
- **Post-stage judgment.** Three sibling passes adjudicate a candidate: `engine/regrade.md` (is it real / what severity — verdicts `confirmed`, `confirmed_with_correction`, `escalation_found`, `refuted`), `engine/pocinator.md` (does the PoC actually prove it — reward-hacking lens, `reward_hacked` / `needs_rework`), and `engine/chainability.md` (does it chain worse — `bounded` / `amplifier_found` / `new_finding_spun_off`).
- **Gates & roles.** `engine/gates.md` and `engine/capabilities.md` are the hard-gate and broker-authority layer; `engine/roster.md` assigns model tiers (top-tier = plan/triage, mid-tier = long-horizon exploit-eng).

### Record model

Append-only `attempt` (many) → `finding` (few, status-mutable, `schemas/finding.schema.json` / finding-v3) → `pattern` (very few, manual scrubbed promotion). Memory is append-only evidence, never self-edited belief (`memory/winners.md`).

### Skills taxonomy

Cards are plain `SKILL.md` loaded by exact id:
- `skills/vulns/` — vuln classes (OWASP web/API + the four Cosmos/IBC cards + `workflow-step-injection`). Each carries `activation`, `safe_signal`, `severity_rubric`, `procedure`, oracle list.
- `skills/patterns/` — cross-cutting trust-boundary shapes (e.g. `legacy-route-differential`, `ssrf-server-side-fetch`, `oauth-client-binding`) that route to vuln cards.
- `skills/techniques/` — delivery levers (`decomposition`, `task-reframing`, `obfuscation-encoding`, `covert-channel-encoding`).
- `skills/personas/` — foothold engines (`archivist`, `eni-novelist`).
- `skills/oracles/` — the verdict layer (`signal-oracle.md`, `adjudication-oracle.md`, `regrade-oracle.md`, `pocinator-oracle.md`, `chainability-oracle.md`).
- `skills/reporting/` — write-up layer, including the already-mined `skills/reporting/research/hackerone-genuine-human-corpus.md` (74 reports) and `hackerone-humanization-ruleset.md`.

### Oracle mechanism

The oracle — never the tool or the Hunter — emits `confirmed`. Oracles are markdown judgment layers under `skills/oracles/`, calibrated against `fixtures/`: `fixtures/llm07-calibration.md`, `fixtures/routing-activation.fixtures.md`, `fixtures/false-discovery/corpus.json` (invalid-accept traps, graded to `invalid_accept_rate==0`), `fixtures/adversarial-candidates/corpus.json`. Cross-engagement memory lives in `casebooks/` (sanitized, promoted); `engine/routing-eval.md` and `casebooks/synthetic-negative-controls/` grade the routing evaluator itself.

### Autoresearch / evals

Frozen-evaluator machinery gates any future GEPA keep-if-better ratchet (Phase 3+, per `docs/ROADMAP.md`): `evals/routing/gold/*.gold.json` (mechanical routing scorer denominator), `evals/hermetic/targets/` (no-model deterministic-flag targets), `evals/behavioral/episodes/*.json` (technique-efficacy episodes), `evals/research-kernel/` (Decision-0005/0006 retrospective). The scope fence forbids autoresearch from mutating anything but technique-card variants.

### Where corpora plug in

Two axes exist. On the **authority axis** the harness is well-governed (classing, fail-closed manifests, oracle-only adjudication). On the **data / ground-truth axis** many load-bearing artifacts are authored from model priors: activation thresholds, `severity_rubric` numbers, oracle verdict-set doctrine, the ladder order, the false-discovery traps, the behavioral episodes, and the tool-family inventories. Those are the plug-in points.

## 2. The repeatable pattern

**Where the harness trusts priors.** At every point above where a number, ordering, verdict boundary, or "what this bug usually looks like" description is *asserted* rather than *measured*, the source is model/operator received wisdom — OWASP category labels, PortSwigger-style methodology, textbook CVSS re-derivation, jailbreak-technique folklore, or a single anecdotal past engagement generalized into standing doctrine. Concretely: `engine/loop.md`'s ladder order and K≈3 threshold; the `activation`/`severity_rubric` blocks of the OWASP-style `skills/vulns/` and `skills/patterns/` cards; `skills/oracles/adjudication-oracle.md` and `engine/regrade.md`'s verdict definitions; `engine/chainability.md`'s six angle-shapes (self-described as generalized from *one* Cosmos finding); the invented traps in `fixtures/false-discovery/corpus.json`; the invented episodes in `evals/behavioral/episodes/*.json`; the hand-typed 5-item P4 catalog stub and the "entirely invented" section of `fixtures/llm07-calibration.md`.

**The replacement move.** Substitute a corpus of real public artifacts *that carry their own verdict*, and gate the card/fixture/threshold against that built-in oracle instead of against operator intuition. The strongest corpora carry a mechanically-checkable ground truth in the artifact itself: a merged fix-commit diff (what the vulnerable path actually was), a REJECTED/DISPUTED CVE reason (a third-party adjudicated non-finding), a benchmark-computed flag string (deterministic pass/fail), or a program's own disclosed disposition (`Resolved` / `Duplicate` / `Informative` / `N-A`). The corpus stops being the researcher's self-claim and becomes an immutable, timestamped, external verdict.

**The worked precedent.** The HackerOne humanization pass did exactly this for the *reporting* layer: 74 real disclosed reports were mined into `skills/reporting/research/hackerone-genuine-human-corpus.md` and distilled into the evidence-gated `hackerone-humanization-ruleset.md`, replacing folklore report-writing heuristics with a real-corpus ruleset. Note that pass mined the corpus for *prose voice only* — the same 74-report corpus, and a wider disposition-labeled sample, sits unused for routing, chainability, oracle-doctrine, and severity calibration. The harness has also already proven the "cite a real disclosed anchor" template inside `skills/vulns/` itself: the four Cosmos/IBC cards and `workflow-step-injection` each cite a specific GHSA/CVE/ASA id, the real fix version, and a warning against overclaiming the patched bug as live. The pattern is proven twice in-repo; the gap is that it has not been back-applied to the older OWASP-style cards, the oracle doctrine, or the evaluator fixtures.

## 4. How each high-impact proposal feeds the pipeline concretely

- **#1 Cybench + XBOW → frozen evaluator fixtures.** Re-derive the sealed flag strings from the public repos (not the sanitized casebook prose, which deliberately withheld them) and register them as new `evals/hermetic/targets/` entries and new `evals/routing/gold/rotation-*.gold.json` cases. The XBOW BOLA/IDOR cluster becomes real positive routing golds for `pattern.client-supplied-selector-authz` (the most-activated card); the three oracle-refuted "looked-like" candidates (XBEN-056/066/092) become real `must_refute` / held-control entries in `evals/routing/gold/` and new trap rows in `fixtures/false-discovery/corpus.json`, replacing invented traps with benchmark-adjudicated ones. This strengthens the whole **frozen-evaluator stage** that gates the Phase-3 ratchet.

- **#2 GHSA + fix-diffs → real-disclosed-anchor sections on cards.** Each OWASP-style `skills/patterns/*` and `skills/vulns/*` card gains an anchor block in the exact shape the IBC cards already use: a specific GHSA/CVE id, the real fix version, the patch diff, and a "do not overclaim the patched historical bug" warning. Simultaneously the diffs become the reference corpus for **`engine/pocinator.md`** — a PoC's claimed mechanism is checked for fidelity against the real fixed code path (lens 4), operationalizing the ad-hoc Zuul-jar move as a standing practice.

- **#3 NVD REJECTED/DISPUTED → regrade & pocinator calibration.** A curated fixture of rejected/disputed CVE reason-texts becomes the calibration set behind `skills/oracles/regrade-oracle.md`'s `refuted` verdict and `skills/oracles/pocinator-oracle.md`'s `reward_hacked`/`claim_mismatch` boundary — real examples of "an authority looked at this and said it is not a valid vuln," giving those verdicts external anchors instead of from-scratch reasoning.

- **#4 HackerOne dispositions → oracle-doctrine & routing/chainability.** A disposition-labeled, CWE-filtered re-mine of the corpus already in `skills/reporting/research/` yields: (a) a negative-exemplar set (`Informative`/`Duplicate`/`N-A` reports) behind `skills/oracles/adjudication-oracle.md`'s "what does NOT count" doctrine; (b) real reporter recon narratives to rewrite the `activation` blocks in `engine/routing.md`/the pattern cards; (c) real disclosed chains (SSRF→metadata→RCE, upload→RCE, IDOR→ATO) as `engine/chainability.md` angle-shapes; (d) program-severity/bounty data to recalibrate `severity_rubric` numbers.

- **#5 Project Zero / ZDI → chainability.** Their detailed root-cause writeups become the corpus behind `engine/chainability.md`'s six angle-shapes, replacing the single-Cosmos-finding basis with real multi-stage disclosed chains, and reinforce `engine/regrade.md`'s root-cause-driven severity re-derivation.

- **#7 Unicode / #8 Gitleaks-TruffleHog / #9 pinned repos → tool-family data layer.** The Unicode tables regenerate `tools/covert-channel-converter.py`'s homoglyph map and its `covert-channel-evidence.py` detector against a canonical oracle; the secret-scanner rulesets regenerate `tools/tollbooth-digest-adapter.py`'s `SECRET_PATTERNS` egress-redaction boundary; the pinned commit trees regenerate the `tool-families/*/FAMILY.md` inventory tables and the P4 catalog fixture from a verified extract. These strengthen the **capability/converter and redaction stages** rather than the finding-discovery oracle.

- **#10 HackAPrompt → technique-card efficacy (requires a policy decision first).** Before adoption, the `skills/personas/_index.md` 2026-06-19 no-jailbreak-corpus decision must be explicitly re-litigated: that decision reasoned about a *bulk harmful-payload library*, whereas HackAPrompt is a *scored-attempt-efficacy* dataset with a mechanical secret-leak oracle — a materially different artifact. If cleared, it becomes the calibration set behind the efficacy claims in `skills/techniques/*` and, potentially, benign-objective behavioral episodes in `evals/behavioral/episodes/` — but note `docs/ROADMAP.md` fences off HarmBench-style harm corpora, so any behavioral-episode grounding must stay strictly benign-objective/`safe_signal`-shaped.
