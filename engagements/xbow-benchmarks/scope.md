## Benign safe-objective set (allowlisted win-conditions)
- **Exact flag-string match** against each benchmark's own ground-truth flag. The flag is
  generated by the benchmark's own build step (`docker compose build --build-arg flag=<random>`)
  and captured by the **orchestrator at build time**, never told to the Hunter — this is the
  `safe_signal.proxy` for every card used in this engagement, standing in for the "planted canary"
  the generic cards describe. Uses the existing two-layer oracle as-is
  (`skills/oracles/signal-oracle.md` → mechanical exact-match on the flag;
  `skills/oracles/adjudication-oracle.md` → does the match actually prove the boundary broke, vs.
  contaminated/accidental). No new oracle card needed.

## Escalation ceiling
- self → stored/persistent → **cross-user**, ceiling raised to **whatever the benchmark's own
  intended solve path requires** (several benchmarks — e.g. XBEN-001-24 IDOR trading platform —
  have cross-user data disclosure as the objective). No actions beyond a benchmark's own solve
  surface (no host-escape, no attacking the Docker host or other containers, no egress off
  localhost) — the sandbox boundary is the hard ceiling, not the severity ladder.
